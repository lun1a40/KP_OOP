# ToyFactoryAssets

Курсовая работа по дисциплине «Объектно-ориентированное программирование».

**Тема:** Реализация объектно-ориентированной парадигмы для учёта данных об управлении основными фондами предприятия по производству детских игрушек.

## Что делает программа

Учёт основных фондов фабрики игрушек:

- **Производственное оборудование** — термопластавтоматы, конвейеры покраски и упаковки
- **Здания** — производственные цеха, склады, административные корпуса
- **Транспорт** — погрузчики, фургоны доставки
- **Офисная техника** — компьютеры, принтеры

Поддерживает:

- Регистрацию, редактирование, списание фондов
- Перемещение между подразделениями (цех → склад, и т.д.)
- Журнал ремонтов и обслуживания
- Расчёт амортизации (разные алгоритмы для разных типов фондов)
- Отчёты: остаточная стоимость на дату, движение за период, разбивка по подразделениям

## Архитектура

C++ сервер на cpp-httplib отдаёт и REST API, и статику фронтенда. Данные хранятся в JSON-файлах. Никаких внешних СУБД, никаких отдельных рантаймов — всё запускается одним бинарём.

См. [docs/architecture.md](docs/architecture.md) для детального описания.

## Запуск

### Через Docker (рекомендуется)

```bash
docker compose up --build
```

Сервер поднимется на `http://localhost:8080`. Данные (`data/`) монтируются с хоста — изменения переживают пересоздание контейнера.

### Локально через CMake

```bash
cmake -S . -B build && cmake --build build -j
./build/ToyFactoryAssets        # запускать из корня проекта (нужны ./web и ./data)
```

### В Visual Studio (Windows)

CMake умеет сгенерировать решение: `cmake -S . -B build -G "Visual Studio 17 2022"`, затем открыть `build/ToyFactoryAssets.sln`.

Подробнее: [docs/build.md](docs/build.md).

## Технологии

- C++17 (портируемый: g++/clang/MSVC)
- [cpp-httplib](https://github.com/yhirose/cpp-httplib) — header-only HTTP
- [nlohmann/json](https://github.com/nlohmann/json) — header-only JSON
- CMake + Docker (multi-stage) — сборка и запуск
- HTML/CSS/JS на фронте (без фреймворков)
